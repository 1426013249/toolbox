# ToolBox 差异化竞争与增长战略规划

## 1. 差异化定位方向选择：推荐 B + A 混合模式
**推荐方向：【隐私安全 + 中国特色痛点】(Privacy-First Local Practical Tools)**

### 为什么选择这个方向？
1. **信任背书（针对 A）**：普通用户（尤其是涉及身份证、简历、敏感截图的用户）对“上传服务器”有天然的戒备心。**“零上传、纯本地”**不仅是技术架构，更是核心营销卖点（Selling Point）。
2. **流量洼地（针对 B）**：PDF 转 Word 等通用需求已被大厂垄断，但**“身份证加水印”、“证件照换底色”、“适配微信限制的图片压缩”**等中国特色细分需求，大厂不屑做、小厂做不好（大多带广告或需强制关注公众号）。
3. **技术可行性**：现代浏览器（WebAssembly, TensorFlow.js）已足以在前端完成 90% 的图像/文件处理任务，Vercel 部署的零服务器架构正好支撑这一特性，实现零成本覆盖海量用户。

## 3. 第一个应该做哪个？
**首推：【证件照 Pro (换底/裁剪/排版)】**

*   **推荐理由**：
    *   **传播潜力**：它是典型的“工具驱动增长”案例。用户在急需报名（考公/考研/入职）时，发现一个免费、好用、不需下载的工具，极易在社群传播。
    *   **差异化**：市面上大多换底色工具都要收“6元下载费”。我们提供纯前端免费版，能迅速积累第一波核心用户。
    *   **开发难度**：已有成熟的 MediaPipe 算力支持，纯前端实现抠图已非常丝滑。
    *   **价值感**：相对于简单的水印，证件照处理具有更高的视觉冲击力和“黑科技”感。

## 4. 执行节奏：先扩充还是先推广？

**当前状态**：4 个工具（PDF/图片/截图/脱敏），尚未形成“百宝箱”规模感。

### 第一阶段：扩充期 (当前 - 1个月)
*   **目标**：工具数量达到 **8-10 个**，补齐“证件照”和“身份证水印”两个核心爆款。
*   **动作**：优化现有工具的 UI 统一性，确保每个工具都有明确的“隐私保护说明”。

### 第二阶段：爆发期 (第 2 个月)
*   **目标**：利用爆款工具进行低成本引流。
*   **动作**：
    *   在小红书、V2EX、即刻、知乎发布“安利”帖，主打**“良心、纯前端、学生党必备”**。
    *   在 GitHub 开源，吸引开发者贡献更多前端工具逻辑。

### 第三阶段：迭代期 (长期)
*   **目标**：形成“处理敏感数据就用 ToolBox”的品牌心智。
*   **动作**：根据用户反馈通过 `/bug` 或 `feedback` 增加新工具，保持每周一个微小更新的节奏。

---

**结论**：不要做“全才”，要做“针对中国普通用户隐私场景的利基工具集”。先用**证件照**和**水印**立住脚跟，再通过**纯前端**的低成本优势打持久战。
